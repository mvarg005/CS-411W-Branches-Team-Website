Glossary

Here are the key terms related to the Branches project:

The process of verifying the identity of a user when they log into the platform, typically using a username and password.

The options users have to control who can see their posts and personal information on the platform.

A page that displays the personal details of a user, such as their name, photo, and list of posts or connections.

The feature that allows users to control who can see their individual posts, such as public, friends, or only me.

A preliminary model of the platform used to test and visualize the functionality and design before full development.

A short description of a feature or requirement from the perspective of an end user, typically used in agile development.

A set period during which specific tasks or features are completed as part of an agile development cycle.

A methodology used for software development where requirements and solutions evolve through collaborative work among cross-functional teams. It promotes flexibility, customer feedback, and rapid delivery of functional software in iterative cycles (called sprints).

A set of protocols, routines, and tools that allow different software applications to communicate with each other. APIs enable the **Branches** platform to integrate with external services (like social media, databases, etc.).

The part of the web application that handles the data, logic, and server-side processes. It includes the server, database, and application logic.

A specific URL where an API can access the resources or data it needs. Each endpoint corresponds to a specific functionality or resource in the API.

An acronym for Create, Read, Update, and Delete – the four basic functions of persistent storage and fundamental to database management and web applications.

The part of the web application or website that users interact with directly. It includes everything the user experiences visually, like layouts, text, images, and buttons.

A set of tools, libraries, and best practices used to develop the backend of a web application. Examples include Express.js (Node.js), Django (Python), and Ruby on Rails.

An architectural style for designing networked applications, using a stateless communication protocol (usually HTTP) and defining a set of standard methods such as GET, POST, PUT, and DELETE.

An architectural design where an application is divided into small, independently deployable services, each performing a specific function. This approach allows for greater flexibility, scalability, and easier maintenance.

A software development practice where code changes are automatically tested and integrated into the main project repository frequently, often multiple times a day, to ensure code quality and reduce integration issues.

A practice where code changes that pass automated tests are automatically deployed to production, enabling fast and reliable delivery of new features and bug fixes.

A system that tracks changes to a codebase, allowing developers to manage different versions of the code and collaborate on projects. Git is one of the most popular version control systems.

A protocol that enables two-way communication between a client and a server over a single, persistent connection. WebSockets are often used for real-time applications like chat apps and live updates.

A platform for developing, shipping, and running applications in containers. Docker allows developers to package software with all its dependencies into a container, making it easier to deploy across various environments.
